# 네비게이션 구조 (Navigation)

## 1. 개요

이 문서는 스포츠스코어픽 앱의 네비게이션 구조와 메뉴 체계(IA)를 정의합니다.

- **원본 PDF 참조**: 4-5페이지 (메뉴구조, 하단 탭 구조)
- **적용 범위**: 앱 전역 네비게이션

---

## 2. 앱 정보 아키텍처 (IA)

### 2.1 메인 탭 구조 (Bottom Navigation)

```
┌──────┬──────────┬──────────┬──────┬────┐
│  홈  │ 픽전문가 │ 커뮤니티 │ 랭킹 │ MY │
└──────┴──────────┴──────────┴──────┴────┘
```

### 2.2 탭별 상세 구조

| 순서 | 탭 이름 | 영문명 | 아이콘 | 주요 기능 |
|------|---------|--------|--------|----------|
| 1 | 홈 | Home | 홈 아이콘 | 라이브스코어, 프로토리그 |
| 2 | 픽전문가 | Pick Expert | 픽 아이콘 | 유료픽, 무료픽, MY픽 |
| 3 | 커뮤니티 | Community | 커뮤니티 아이콘 | 게시글, 댓글, 좋아요 |
| 4 | 랭킹 | Ranking | 랭킹 아이콘 | 유저 랭킹, 순위 |
| 5 | MY | MY | 프로필 아이콘 | 마이페이지, 설정 |

## 3. 탭별 하위 메뉴 구조

│   ├── 전체리그

## 4. 네비게이션 동작

### 4.1 탭 전환

| 동작 | 설명 |
|------|------|
| 탭 선택 | 해당 탭의 루트 화면으로 이동 |
| 현재 탭 재선택 | 스크롤 최상단으로 이동 또는 루트 화면으로 복귀 |
| 탭 간 이동 | 각 탭의 스택 상태 유지 |

### 4.2 화면 전환

| 전환 타입 | 사용 케이스 | 애니메이션 |
|----------|------------|------------|
| Push | 상세 화면 진입 | 우측에서 슬라이드 |
| Pop | 이전 화면 복귀 | 좌측으로 슬라이드 |
| Modal | 모달/팝업 | 하단에서 슬라이드 업 |
| Replace | 화면 교체 | 페이드 또는 없음 |

### 4.3 딥링크 지원

| 경로 | 화면 |
|------|------|
| /home | 홈 |
| /home/match/:id | 경기 상세 |
| /pick | 픽전문가 |
| /pick/:id | 픽 상세 |
| /community | 커뮤니티 |
| /community/post/:id | 게시글 상세 |
| /ranking | 랭킹 |
| /ranking/user/:id | 유저 프로필 |
| /my | 마이페이지 |

## 5. 헤더 (App Bar) 구성

### 5.1 메인 탭 헤더

```
┌──────────────────────────────────────┐
│ [로고]                    [검색][알림]│
└──────────────────────────────────────┘
```

### 5.2 상세 화면 헤더

```
┌──────────────────────────────────────┐
│ [←]  [타이틀]               [옵션...]│
└──────────────────────────────────────┘
```

### 5.3 헤더 구성 요소

| 요소 | 위치 | 동작 |
|------|------|------|
| 뒤로가기 (←) | 좌측 | 이전 화면으로 복귀 |
| 타이틀 | 중앙 | 현재 화면명 표시 |
| 검색 | 우측 | 검색 화면 진입 |
| 알림 | 우측 | 알림 목록 진입 |
| 옵션 (...) | 우측 | 추가 메뉴 표시 |

## 6. 데이터 요구사항

### 6.1 네비게이션 상태

```typescript
interface NavigationState {
  currentTab: 'home' | 'pick' | 'community' | 'ranking' | 'my';
  tabStacks: {
    home: Screen[];
    pick: Screen[];
    community: Screen[];
    ranking: Screen[];
    my: Screen[];
  };
}

interface Screen {
  route: string;
  params?: Record<string, any>;
  scrollPosition?: number;
}
```
