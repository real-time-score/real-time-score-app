# 프로토 화면 구성

## 1. 개요

### 1.1 목적
프로토 화면은 사용자가 공식 스포츠 복권 서비스(Big, toto, winner)에 쉽게 접근할 수 있도록 외부 링크를 제공하는 화면입니다.

### 1.2 원본 참조
- **PDF 페이지**: 43-44

---

## 2. 화면 구성

### 2.1 화면 위치
- **진입 경로**: 홈 탭 > 상단 탭 메뉴에서 "프로토" 선택
- **탭 구조**: `LIVE` | `프로토` (프로토 선택 시 해당 화면 표시)

### 2.2 레이아웃 구조

```
┌─────────────────────────────────────┐
│ リアルタイムスコア              🔔  │  ← 헤더 (일본어 서비스명)

│       [배너 이미지 영역]      1/3   │  ← 배너 슬라이더

│    LIVE        [프로토]             │  ← 탭 (프로토 활성 - 네이비 배경)
├─────────────────────────────────────┤
│                                     │
│  ┌───────────────────────────────┐  │

│  │ 최대 6억 엔에 도전하세요!      │  │  ← Big 배너 (이미지 + 텍스트)
│  │ キャンペーン期間: 2025년11月15日│  │
│  └───────────────────────────────┘  │

│  │ 실력형 스포츠 복권             │  │
│  └───────────────────────────────┘  │
│                                     │
│  ┌───────────────────────────────┐  │
│  │ 하나의 경기, 하나의 선택!      │  │
```

> **디자인 참고**: 프로토 탭 활성 시 네이비 배경+흰색 텍스트, Big 배너는 이미지 배경에 텍스트 오버레이

### 2.3 주요 컴포넌트

| 컴포넌트 | 설명 |
|---------|------|
| 탭 메뉴 | LIVE/프로토 전환 탭 |
| Big 카드 | Big 서비스 배너 및 설명 |
| toto 카드 | toto 서비스 배너 및 설명 |
| winner 카드 | winner 서비스 배너 및 설명 |

### 4.1 프로토 서비스 정보

```typescript
interface ProtoService {
  id: string;                    // 서비스 고유 ID
  name: string;                  // 서비스명 (Big, toto, winner)
  description: string;           // 서비스 설명

  externalUrl: string;           // 외부 링크 URL
  displayOrder: number;          // 표시 순서
  isActive: boolean;             // 활성화 여부
  createdAt: DateTime;           // 생성일시
  updatedAt: DateTime;           // 수정일시
}
```

## 6. 관리자 기능

### 6.1 프로토 서비스 관리

| 기능 | 설명 |
|------|------|
| 조회 | 등록된 프로토 서비스 목록 확인 |
| 수정 | 서비스 설명, 배너 이미지, 외부 URL 변경 |
| 순서 변경 | 표시 순서 조정 (드래그 앤 드롭) |
| 활성화/비활성화 | 개별 서비스 노출 여부 설정 |

### 6.2 Admin API

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/admin/proto/services` | 서비스 목록 조회 |
| PUT | `/api/admin/proto/services/{id}` | 서비스 정보 수정 |
| PATCH | `/api/admin/proto/services/{id}/status` | 활성화 상태 변경 |
| PUT | `/api/admin/proto/services/order` | 표시 순서 변경 |

### 6.3 관리 화면 요구사항

1. **서비스 목록 화면**
   - 테이블 형태로 서비스 목록 표시
   - 배너 이미지 썸네일 미리보기
   - 활성화 토글 스위치
   - 드래그로 순서 변경

2. **서비스 수정 모달**
   - 서비스명 (읽기 전용)
   - 설명 텍스트 수정
   - 배너 이미지 업로드/변경
   - 외부 URL 수정
   - 저장/취소 버튼

## 7. 구현 시 고려사항

### 7.1 외부 링크 처리
- 인앱 웹뷰 vs 외부 브라우저 선택 가능하도록 구현
- 딥링크/유니버셜 링크 지원 여부 확인 필요

### 7.2 배너 이미지
- 권장 비율: 16:9 또는 2:1
- 최소 해상도: 720 x 360px
- 지원 포맷: JPG, PNG, WebP

### 7.3 캐싱 전략
- 서비스 정보는 변경 빈도가 낮으므로 로컬 캐싱 권장
- 앱 시작 시 또는 pull-to-refresh 시 갱신
