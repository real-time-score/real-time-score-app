## 모바일 앱 UI 개발 가이드

### 기본 원칙
- **디자인 일치 최우선**: 전달받은 페이지(디자인)와 일치하게 UI를 구현하는 것이 최우선 목표
- 기본적으로 프로젝트에 정의된 타이포그래피, 폰트, 컴포넌트를 활용하여 작업

### 컴포넌트 사용 기준
1. **컴포넌트 사용 가능한 경우**: 정의된 컴포넌트로 디자인과 일치하게 구현 가능하면 컴포넌트 사용
2. **컴포넌트 사용 불가능한 경우**: 아래 상황에서는 컴포넌트를 사용하지 않고 직접 구현
   - 컴포넌트가 해당 디자인에 적합하지 않을 때
   - 컴포넌트 사용 시 디자인과 일치하게 구현이 불가능할 때
   - 컴포넌트 커스터마이징이 과도하게 필요할 때

### 우선순위
```
디자인 일치 > 컴포넌트 재사용
```
- 컴포넌트 재사용보다 **디자인 일치도**가 더 중요
- 억지로 컴포넌트에 맞추느라 디자인을 변형하지 말 것
